### Features Considered:
1. **mx missile**: Vote on MX missile (2 = yes / 1 = no)
2. **aid to nicaraguan contras**: Vote on aid to Nicaraguan contras (2 = yes / 1 = no)
3. **handicapped infants**: Vote on the issue of handicapped infants (2 = yes / 1 = no)
4. **superfund right to sue**: Vote on superfund right to sue (2 = yes / 1 = no)

### Decision Tree Structure:
- The decision tree is structured to evaluate each feature in sequence.
- Each node checks if the respective feature value is at least 2 (indicating a 'yes' vote).
- If all conditions are met, the prediction is Republican (1); otherwise, it's Democrat (0).

### Approach:
1. **Evaluate Conditions**: Compute whether each feature meets its threshold and store these results.
2. **Determine Prediction**: Use nested if-else statements based on the evaluated conditions to decide the party affiliation.

```python
def predict(X: dict):
    nodes = [None] * 4
    
    # Evaluate each node condition
    nodes[0] = (X['mx missile'] >= 2)
    nodes[1] = (X['aid to nicaraguan contras'] >= 2)
    nodes[2] = (X['handicapped infants'] >= 2)
    nodes[3] = (X['superfund right to sue'] >= 2)
    
    # Convert boolean values to integers for the nodes list
    nodes_int = [int(node) for node in nodes]
    
    if nodes[0]:
        if nodes[1]:
            if nodes[2]:
                prediction = 1 if nodes[3] else 0
            else:
                prediction = 0
        else:
            prediction = 0
    else:
        prediction = 0
    
    return prediction, nodes_int
```

### Explanation:
- **Nodes Evaluation**: Each node condition is checked in sequence. The results are stored as boolean values.
- **Prediction Logic**: Starting from the first node, each subsequent node is only evaluated if the previous conditions were met. If any condition fails (i.e., a feature doesn't meet its threshold), the prediction defaults to Democrat (0).
- **Nodes List**: Converted to integers where `True` becomes 1 and `False` becomes 0, ensuring clarity in which conditions were satisfied.

This approach ensures that all inner node evaluations are captured, providing transparency into the decision-making process of the tree.
